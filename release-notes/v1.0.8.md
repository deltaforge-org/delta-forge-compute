## What's Changed

The DeltaForge compute node: the query-executing process. It connects to a DeltaForge control plane and runs queries on a single node. Scale out by running more nodes.

### Direct-dial query routing

- Clients (drivers, CLI, GUI) now dial a compute node directly and receive results as a compact length-prefixed binary frame format. The control plane is a directory that hands out node addresses, not a data-path proxy, so query results no longer round-trip through it.
- Every client request carries a uniform `x-deltaforge-node` selector header. Set it to target a specific node; leave it empty and the server picks one, balancing load across the pool. One client code path serves both direct-dial and a routing gateway.

### Single-node cloud deployment

- A compute node can be deployed as a single signed binary alongside a platform node on Azure, AWS, or GCP, using native per-cloud infrastructure-as-code.

### Stability and engine improvements

- `SHOW STATS ACTUAL` now surfaces disk-spill activity when a query spills to disk during execution.
- More robust auto-discovery for JSON-family files and FHIR NDJSON.
- Fixed fresh-bootstrap authentication errors; stale logins now self-heal.
